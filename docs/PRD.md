## 1. 背景与目标

### 1.1 背景
当前主流文件处理工具(在线压缩、在线 PDF 合并、在线视频转码等)存在三大痛点:

1. **隐私风险**:用户必须把私人照片、合同、视频上传到陌生服务器,无法确认是否被留存、训练或泄露。
2. **功能割裂**:图片工具、PDF 工具、视频工具各自为政,用户需在多个网站/软件间反复切换,且每个都有体积上限、水印、付费墙。
3. **批量能力弱**:大多数在线工具只支持单文件或少批量,处理几百个文件时效率极低且容易中断。

### 1.2 目标
做一个**一站式、本地运行、批量处理、隐私安全**的桌面工具,让用户在一个应用内完成绝大多数日常文件处理需求,**全程不离机**。

### 1.3 核心价值主张
- 🖥️ **本地优先**:所有处理在用户设备本地完成,文件内容绝不离开本机。
- 🔒 **隐私可证**:开源可审计,默认零遥测,日志/临时文件可一键清除。
- ⚡ **轻量高效**:安装包几 MB,占用低,多核并行处理大批量文件。
- 🧰 **一站聚合**:图片/文档/视频/音频/文件管理,一个应用替代一抽屉小工具。
- 🔗 **流水线串联**:支持把多个操作串起来一次执行(核心差异化能力)。
- 🔄 **安全可控**:操作前预览、操作后可撤销,失败不中断。

### 3.2 详细功能描述

#### 3.2.1 图片处理

**批量压缩 / 格式转换(MVP)**
- 输入:支持 JPG/PNG/WebP/GIF/BMP/TIFF 等常见格式批量导入。
- 输出:可指定目标格式(JPG/PNG/WebP/AVIF)与压缩质量(0–100)。
- 支持保持原始目录结构输出,或统一输出到指定目录。
- 提供处理前后**体积对比**与**画质预览**。

**调整尺寸 / 批量裁剪(MVP)**
- 按固定尺寸、按比例缩放、按最长边/最短边约束。
- 批量裁剪支持统一边距、居中裁剪。

**EXIF 处理(隐私卖点)**
- 默认去除 EXIF(位置、设备、时间),可切换为"保留"。
- 在 UI 中明确提示"已去除 N 项隐私信息"。

**目标体积压缩**
- 用户指定"压到 200KB 以内",工具自动迭代调整质量参数命中目标。

**批量加水印**
- 文字水印(字体/颜色/透明度/位置)或图片水印,支持九宫格定位。

#### 3.2.2 文档处理(PDF / Office)

**PDF 合并 / 拆分(MVP)**
- 合并:拖入多个 PDF,可拖拽调整顺序,输出单一文件。
- 拆分:按页码区间、按每 N 页、按书签拆分。

**PDF 压缩(MVP)**
- 通过重采样图片、子集化字体、压缩流降低体积。
- 提供压缩前后体积对比。

**页面操作**:旋转、删除、提取、重排、插入空白页。
**互转**:PDF→图片(可设 DPI)、图片→PDF;PDF→Word/HTML(依赖 LibreOffice)。
**加密 / 权限**:设置打开密码、禁止复制/打印权限。
**OCR**:集成 Tesseract,支持中英文,离线运行,输出可搜索 PDF。

#### 3.2.3 视频处理

- 底层统一调用 **ffmpeg**,以子进程方式调用。
- **剪切**:精确到帧的时间段裁剪,支持无损复制流(`-c copy`)快速剪切。
- **转码**:常见格式互转(MP4/MOV/MKV/WebM),编码可选 H.264/H.265/AV1。
- **硬件加速**:自动检测 GPU(NVENC / QSV / VideoToolbox),可一键启用。
- **压缩**:目标码率或目标体积压缩。
- **音频提取 / GIF**:从视频提取音轨、截取片段生成 GIF。
- **字幕**:烧录字幕到画面、或提取内嵌字幕。

#### 3.2.4 音频处理

- 格式转换(MP3/AAC/FLAC/WAV/OGG)、剪切合并、音量标准化(`loudnorm` 滤镜)。
- 复用视频模块的 ffmpeg 调用栈。

#### 3.2.5 文件管理

**批量重命名(MVP)**
- 支持模板变量:`{name}` `{ext}` `{date}` `{index:3}` `{exif.date}` 等。
- 实时**预览**新文件名,支持冲突检测。
- 支持 Exif 拍摄日期、文件创建/修改时间作为命名来源。

**重复文件查重(MVP)**
- 两阶段:先按**大小**预筛 → 再按**内容哈希**(`blake3`)精确比对。
- 支持按文件夹范围扫描,结果分组展示,可勾选批量删除/移动。
- 提供"保留最新/最大/第一个"等智能保留策略。

**批量解压 / 压缩**:zip/7z/rar/tar.gz 批量解压;批量打包并可选密码。
**文件校验**:计算 MD5/SHA1/SHA256,支持与预期值比对。
**磁盘占用可视化**:扫描目录,按体积生成树状/旭日图。

#### 3.2.6 ★ 流水线(核心差异化能力)

> 把多个原子操作串成一条流水线,对一批文件一次性完成"压缩 → 重命名 → 打包"等多步处理。

- **可视化编辑器**:节点式 UI,每个节点 = 一个操作;支持拖拽编排、分支、并行。
- **节点类型**:图片压缩、格式转换、重命名、解压、查重、自定义脚本……
- **流水线模板**:内置常用模板("照片整理归档"、"合同 PDF 标准化"),支持保存/导入导出(JSON)。
- **变量传递**:上游节点的输出可作为下游输入(如提取的 EXIF 日期供重命名节点使用)。
- **预览与执行**:执行前展示完整处理计划(dry-run),执行中实时进度,可暂停/取消。

## 4. 非功能需求

### 4.1 性能
- 批量处理默认**多核并行**(`rayon`),核数可配置。
- 大文件采用**流式处理**,避免一次性载入内存。
- 图片压缩/查重等 CPU 密集任务,处理 1000 张普通图片应在分钟级完成。

### 4.2 可靠性与安全(批量操作红线)
- **失败隔离**:单个文件失败不中断整批,记录到错误日志,可重试单条。
- **Dry-run 预览**:破坏性操作(重命名、删除、覆盖)执行前必须预览结果。
- **撤销栈**:重命名、删除等操作维护可撤销历史(优先用"移入回收站"而非直接删除)。
- **可取消**:任意任务可随时取消,已处理部分保持一致状态。
- **不覆盖**:默认输出到独立目录或加后缀,避免覆盖原始文件;覆盖需显式确认。

### 4.3 隐私与安全
- 🔒 文件内容**绝不**上传网络,所有处理本地完成。
- 🔒 默认**零遥测**;崩溃日志收集须显式同意、可关闭。
- 🔒 临时文件/日志存放于用户私有目录,提供**一键清除**。
- 🔒 不引入强制联网依赖,核心功能**离线可用**。
- 🔒 EXIF/元数据默认提供去除选项。

### 4.4 可移植与分发
- 跨平台:Windows / macOS / Linux 三端原生体验。
- 提供**便携版**(portable),免安装,降低用户对"装了什么"的顾虑。
- 安装包体积控制在合理范围(核心包尽量小,重依赖按需下载)。

### 4.5 可访问性与国际化
- 支持简体中文 / 英文双语,UI 文案外置便于扩展。
- 遵循系统深/浅色主题。
- 键盘可达,核心操作有快捷键。

## 5. 技术架构(概要,详见 `docs/design/`)

### 5.1 技术选型

| 层级     | 选型                                 | 理由                                       |
| -------- | ------------------------------------ | ------------------------------------------ |
| 桌面框架 | **Tauri 2**                          | 体积小(几 MB)、内存低、契合"轻量本地"定位 |
| 后端语言 | **Rust**                             | 性能强、内存安全、并发模型适合批量处理      |
| 前端     | Vue 3 + TypeScript                 | 组合式 API,生态成熟,团队已有 Vue 基础      |
| 图片     | `image` crate / `libvips` 绑定       | 高性能图片处理                             |
| 视频/音频 | **ffmpeg**(子进程调用)             | 行业标准,所有语言通用                      |
| PDF      | `lopdf` / `qpdf` / `mupdf` 绑定      | 合并/拆分/压缩                             |
| 哈希查重 | `blake3`                              | 极快,适合大文件                           |
| Office/OCR | LibreOffice headless / Tesseract    | **可选模块**,按需下载                      |

### 5.2 模块划分

```
src-tauri/src/
├── commands/    # 暴露给前端的 Tauri 命令(每功能一模块)
│   ├── image.rs / pdf.rs / video.rs / rename.rs / dedup.rs
├── pipeline/    # ★ 流水线引擎(节点编排 + 执行)
├── worker/      # 任务队列、进度上报、取消控制
└── common/      # 共享类型、错误、工具函数
```

### 5.3 关键设计点
- **前后端通信**:前端通过 Tauri `invoke` 调用 Rust 命令;长任务用**事件流**推送进度。
- **任务模型**:统一抽象为 `Task`(输入、操作、输出、状态),交给 `worker` 调度执行。
- **可选依赖管理**:ffmpeg/LibreOffice/Tesseract 在首次使用时检测,缺失则引导下载或定位。
